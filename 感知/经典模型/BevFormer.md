​**​空间交叉注意力（SCA）对每一帧独立计算​**​，多帧之间无参数共享或特征交互
先通过TSA融合时间信息，再用更新后的Query执行SCA融合空间信息，BEV Query作为​**​信息载体​**​，在编码器层中交替融合时空特征，逐步细化BEV表示
```mermaid
graph LR
A[输入: BEV Query Q] --> B[TSA: 融合历史BEV特征]
B --> C[更新Query: Q_TSA]
C --> D[SCA: 融合当前帧图像特征]
D --> E[输出: 更新的BEV特征]
```

##### 空间交叉注意力计算（以一个query为例，忽略多头）
1. ​**​生成3D参考点​**​：每个BEV体素位置中心点在自车坐标系下的坐标
2. ​**​投影与视图选择​**​：将4个3D点投影到6个相机，保留有效命中的视图 Vhit​（如3个视图）
3. ​**​动态采样预测​**​：用MLP处理Query向量，预测一个维度为8x2+8+4的向量，分别表示8个xy方向上的偏移量+8个权重+4个尺度的概率
4. ​**​多视图特征采样与插值​**​：在每一个视图上，对于预测概率最大的尺度图层，基于3D参考点投影后的uv位置，应用偏移量和权重，加权得到注意力结果。这一点跟Deformable DETR采用的多尺度并行查找，统一加权是不同的。
5. ​**​多视图聚合​**​：​3个视图的均值作为最终的交叉注意力查询结果，V=(V1​+V2​+V3​)/3

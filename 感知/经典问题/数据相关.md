#### KITTI数据集组织形式的局限性以及NuScenes的改进点
###### KITTI数据集
- 文件名即索引：简单直观、理解成本低
- 时序关系弱：如果你想找“上一帧”，你只能假设上一帧的文件名是 `当前文件名 - 1`。如果中间丢帧了，或者数据集不是连续采集的，这种逻辑就会出错
- 传感器扩展难：KITTI 只有两个主要视角（前视相机+LiDAR）。如果像现在的车一样有 6 个相机、5 个雷达，且频率不同（相机 30Hz，LiDAR 10Hz），靠文件名强行对齐会产生大量冗余或空缺文件
- 类似Excel表
###### NuScenes数据集
- 类似MySQL的关系型数据库设计，每个一类数据（一个json文件）都是一张表。token就是表的主键列。表项之间通过主键建立自由关联，灵活程度高
- sample.json中的prev next指针在关键帧空间进行索引；sample_data.json则在原始帧空间进行索引


#### 车端视频流图像格式
| **环节**        | **数据格式**              | **承载硬件**        | **目的**             |
| ------------- | --------------------- | --------------- | ------------------ |
| **传感器端**      | **RAW (Bayer)**       | CMOS 晶圆         | 原始光电转换             |
| **ISP 后/链路上** | **YUV (4:2:2/4:2:0)** | ISP / SerDes 芯片 | 节省传输带宽，适配人眼视觉      |
| **域控内存中**     | **YUV**               | SoC 内存 (Buffer) | 待处理的原始帧缓存          |
| **模型入口前**     | **RGB Tensor**        | 硬件加速器 (VIC/GPU) | 匹配模型权重布局           |
| **落盘存盘**      | **H.265 / HEVC**      | 硬件编码器 + SSD     | **数据闭环：长期存储与离线训练** |
